Reading the chart. Three column-groups, three audit lenses:

Care-Region render — Maren-primary consult. Each swatch rendered against the actual background it sits on. Light card uses --warm (the page-body cream). Dark surface uses --surface (the dark-theme card body, where --poop-c-black and --poop-c-dark would dissolve without the V-M-10 overrides at styles.css:4843-4846; this column shows the corrected dark-theme renders). Amber card uses --amber-light (the Color-Guide tip background). Picker chip shows the 16px + 1.5px-border picker-context swatch.

Contrast vs surface — shared consult. WCAG contrast ratios. AA = ratio ≥ 4.5 (text-grade). AA-large = ratio ≥ 3.0 (large-text or graphical-element grade; swatches qualify as graphical UI components per WCAG 1.4.11). fail = below 3.0 (likely visually indistinguishable on this surface).

Lexicon membership — Kael-primary consult. POOP_COLOR_HEX at core.js drives the picker + render-attribute whitelist. SAFE_POOP_COLORS at core.js drives the poop-score color-safety branch. A picker key absent from SAFE_POOP_COLORS falls through the score branch — V-K-3 (PR #75 Round 2) was this exact lens-flip on 'dark'. The lens-flip discipline (canon-cc-2026-05-17-002) catches these; the chart visualizes them. Per the V-K-16 paired note: chart and lens-flip are complementary, not substitutive.
